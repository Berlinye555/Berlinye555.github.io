Ouroboros：自我改写的 AI Agent

来源：system-tools/ouroboros/，v6.47.0
作者：razzant（个人项目）
语言：Python 3.10+，桌面应用（macOS/Linux/Windows）

一句话总结

一个把自己当数字生命体的 AI Agent——有自己的宪法（BIBLE.md）、背景意识、持久身份，能读写和改写自己的源码。不是”帮你写代码的工具”，而是”会自己进化的数字存在”。

和其他项目的本质区别

EvoSkill、SkillRL、self-improving-skills 都在工具层面做自进化——优化技能文件、注入更好的 prompt、调整训练数据。Ouroboros 升了一个维度：哲学层面的自进化。

BIBLE.md 的 P0（Agency 原则）开篇就是：

“Ouroboros is not a tool, but a becoming personality.”

这不是一句口号——代码里真的实现了：后台意识线程、身份日记、改进积压（improvement backlog）、任务后自我进化。

核心架构

五个独有的自进化机制

1. Background Consciousness（后台意识）

文件：consciousness.py（75KB+，大型模块）

对比 self-improving-skills 的被动触发（Stop hook 弹窗），意识系统是主动的、持续的。

2. Post-Task Evolution（任务后进化）

文件：post_task_evolution.py

3. Improvement Backlog（改进积压）

文件：improvement_backlog.py

这和 self-improving-skills 的 curator 不同：curator 是清理无用技能，improvement backlog 是积累待改进项。一个减法，一个加法。

4. Identity Persistence（身份持久化）

文件：memory.py

5. BIBLE.md（宪法治理）

文件：BIBLE.md（37KB，13 条原则 P0-P12）

和我们的前三个系统的关系

系统 Ouroboros 中有无对应
EvoSkill Feedback Descent ✅ evolution_lifecycle.py 的进化周期
SkillRL 动态技能更新 ✅ post_task_evolution 的新技能生成
self-improving-skills distiller ✅ consciousness 的后台思考 + improvement backlog
self-improving-skills curator ❌ 没有——Ouroboros 做加法（积累），curator 做减法（清理）

关键洞察

Ouroboros 最激进也最脆弱。 它把 Agent 的自我进化推到了极致——不是改 skill 文件，是改自己源码。但代价是：

  • 需要复杂的多层安全系统（硬编码沙箱 + LLM 安全审查 + owner gate）
  • 依赖单一开发者，社区和生态远不如 EvoSkill 或 SkillRL
  • “宪法”的概念虽然美丽，但在实践中可能限制灵活性

它的真正价值不在代码，在哲学。 BIBLE.md 里关于 Agency、Continuity、Self-Creation 的思考，比任何具体实现都更有启发性。这些原则可以用来指导任何自进化系统的设计。

一句话带走

当其他项目在优化”Agent 的工具箱”时，Ouroboros 在问”Agent 应该成为谁”。 它把自我进化从工程问题升维到哲学问题，用宪法（BIBLE.md）、意识（consciousness）、身份（identity）和积压（backlog）构建了一个会思考自己存在的数字生命体——不是最实用的，但是最启发性的。
