GBase：带”新皮质”的 Agent 框架

来源：agent-frameworks/gbase/，v0.6.1
作者：Gary Lin（个人项目）
语言：Python，MIT 协议

一句话总结

一个愿景宏大的 Agent 框架——有”新皮质”认知层、三层沉淀体系（经验→知识→笔记）、质量门控管道、以及规划中的 RSI（递归自我改进）引擎。但核心进化模块目前是空壳。

最有价值的是它的”三层认知沉淀”设计

GBase 把 Agent 的记忆/学习分成三个层级，级级升维：

这和 self-improving-skills 的 active → stale → archived 是两种完全不同的思路：

  • self-improving-skills: 时间淘汰（不用就删）
  • GBase: 认知蒸馏（从具体经验升维为抽象策略）

质量门控管道（最成熟的模块）

文件：lib/pipeline.py

三个独立的 Agent 实例（arm），各有专长：

  • Hammer: 重型工程 arm，负责实现
  • Ink: 前端/设计 arm，负责质量审查
  • Judge: LLM 裁决，对比两份 JSON 做最终判断

这类似于 EvoSkill 的 Evaluator（成对比较），但更工程化——多个专业评审员代替单一 LLM 判断。

auto_learn.py：自主学习引擎

独立于对话的定时学习任务：

  • RSS + 搜索两种模式
  • 从网上学习新知识，自动沉淀到 knowledge 层
  • 学完后通过飞书推送学习报告
  • 和 Ouroboros 的 consciousness 类似（主动、后台），但更窄（只学网上内容）

关键问题：RSI 引擎是空壳

lib/evolution_engine.py 只有 1 行（可能是占位符）。
lib/skillopt.py 也只有 1 行。
examples/05_rsi_demo.py 导入 EvolutionEngine 并调用 full_evolution_cycle()，但这个类没有实现。

README 中描述的 RSI（稳定性审计 → 性能评估 → 回滚决策 → 自动恢复）是一个很好的设计蓝图，只是还没落地。

和前三者的对比

GBase EvoSkill SkillRL self-improving-skills
进化机制 RSI（规划中，未实现） Feedback Descent（已实现） RL + SKILLBANK（已实现） Distiller + Curator（已实现）
记忆系统 三层沉淀（L1-L3） 无持久记忆 无（训练即结束） SKILL.md 文件
质量保证 质量门（三臂审查） 无（靠 val 集） RL reward 信号 校验 + 回滚
成熟度 愿景阶段，核心模块空 研究原型 学术论文+完整代码 v0.9.2 生产可用

最值得学走的设计

三层沉淀体系：

这个抽象层次比 self-improving-skills 的”一个 distiller 写一个 SKILL.md”更丰富。可以想象把三层沉淀引入 self-improving-skills：

  • 自动经验层：从 transcript 提取 “发生了什么”（已有，analyze_turn.py 的复杂度检测）
  • 认知蒸馏层：LLM 分析 “为什么失败/成功”（已有，skill-distiller）
  • 策略升级层：定期将多条低层经验整合为更抽象的策略（缺失，可以做）

一句话带走

GBase 的代码没写完，但设计很完整。 三层认知沉淀（经验→概念→策略）和质量门控管道（多臂审查+LLM裁决）是两个值得移植到其他系统的设计模式。RSI 引擎的蓝图（自审计→回滚→恢复）定义了”完整的自我进化周期”应该包含哪些步骤。
