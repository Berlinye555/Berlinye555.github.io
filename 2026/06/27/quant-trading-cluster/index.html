量化交易集群：自进化 Agent 在金融领域的应用

来源：quant-trading/ 下 5 个项目

五个项目一句话

项目 一句话 论文 自进化机制
AgentQuant ReAct Agent 循环做策略研究，SQLite 记忆跨 regime 复用 无 regime-aware memory recall
ClawQuant 文件驱动的多语言（TS+Py+Rust）交易 Agent，目标是从每笔交易中学习 无 Trade Learning Record → replay → promote
fincrew 5 个专业 Agent（信息/宏观/技术/复盘/经理）协作做投资决策 无 交易复盘 → 经验沉淀到记忆
open-quant-agent (Robin) 因子研究管线：提出→辩论→实现→验证→融合→策略→回测 无 JSONL 实验记录 + Memory layer
Agent-Trading-Arena 封闭交易模拟环境，研究 LLM 的数字推理能力 EMNLP 2025 + ICML 2026 Self-Play 进化（DecoupledMarket）

三个层次

这 5 个项目不是平行的，它们覆盖了量化+自进化的三个层次：

层次 1：模拟研究（最学术）

Agent-Trading-Arena — 可控市场环境 + Self-Play 进化

  • EMNLP 2025：研究 LLM 的数值理解能力
  • ICML 2026（DecoupledMarket）：在数字孪生市场中通过 Self-Play 进化量化推理
  • 和 SkillRL 最相似——都是学术论文+完整代码+benchmark 驱动

层次 2：研究管线（最工程）

Robin (open-quant-agent) — 因子研究的自动化管线

  • 7 步流水线：Propose → Debate → Implement → Validate → Fuse → Strategy → Backtest
  • Session Host 包装每次运行，记录所有中间产物
  • 类似 EvoSkill 的 Feedback Descent：多步评估 → 择优

AgentQuant — ReAct 循环 + regime 记忆

  • LLM → Grid Search → Random fallback 三级策略生成
  • 锦标赛回测（Sharpe/Calmar/Sortino/Bootstrap）
  • SQLite 记忆：下次遇到相似 regime 召回之前的最佳策略

层次 3：实盘交易（最落地）

ClawQuant — 目标是从每笔交易中学习

  • 文件驱动：markdown 定义策略，JSON 定义配置，JSONL 存储对话
  • Trade Learning Record：每笔平仓交易生成学习记录
  • 两步 permission：Normal mode 沙箱 → Evolution mode 可改自己源码
  • 和 Ouroboros 的”改自己源码”类似，但加了一层 Evolution mode 开关

fincrew — 5 Agent 协作

  • Financial Manager 协调 4 个专业分析员
  • Reviewer Agent 负责交易后复盘
  • 最接近 Hermes 的 multi-agent 编排模式

和 agent-frameworks 集群的关系

量化交易项目本质上是把自进化框架应用到金融领域，但没有一个直接使用了我们前面分析的那些通用框架——它们各自实现了自己的自进化机制。

最值得关注的：Agent-Trading-Arena 的 Self-Play

ICML 2026 的 DecoupledMarket 论文提出了 Self-Play 进化量化推理：

  • 在数字孪生市场中让 Agent 互相博弈
  • 通过自对弈（Self-Play）提升推理能力
  • 这和 SkillRL 的 RL 训练是同一范式（环境→行为→奖励→进化），只是环境从 ALFWorld/WebShop 变成了金融市场

一句话带走

量化交易是自进化 Agent 最自然的应用领域——交易天然有明确反馈（盈亏），天然需要从历史中学习，天然适合”假设→验证→记忆→改进”的循环。 这 5 个项目各自实现了不同粒度的自进化：从学术 Self-Play 到工程化因子管线到实盘交易学习。
