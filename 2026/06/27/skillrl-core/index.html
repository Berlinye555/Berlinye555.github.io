SkillRL：用 RL 训练技能感知的 Agent

来源：agent-frameworks/SkillRL/，论文 arXiv 2602.08234
团队：aiming-lab（NTU 南洋理工等）
代码语言：Python，基于 verl（veRL-agent）+ GRPO 训练器

一句话总结

把 Agent 的历史轨迹（成功+失败）蒸馏成结构化的 JSON 技能库（SKILLBANK），在 RL 训练时将相关技能注入 prompt，并在验证失败时动态生成新技能。技能是纯数据，不是代码。

2. Skill Retrieval（技能检索）

文件：agent_system/memory/skills_only_memory.py（552行）

两种检索模式：

模式 机制 延迟 精度
Template 关键词匹配任务类型 → 返回该类型全部技能 零延迟 召回率高，可能有噪音
Embedding Qwen3-Embedding-0.6B 编码任务描述 → 余弦相似度排序 → top-k 需预计算 embedding 更精准，跨类别搜索

Template 模式的关键细节：动态技能（dyn_NNN）始终全量返回，不会被 top_k 截断——因为它们是针对最新失败生成的，最需要被看到。

Embedding 模式搜索所有类别的 task_specific skills，不限于检测到的任务类型。

3. Dynamic Skill Update（动态技能更新）

文件：agent_system/memory/skill_updater.py（214行）

关键设计：

  • 新技能 ID 由代码分配（dyn_{next_idx:03d}），不信任 LLM 返回的 ID——防止碰撞
  • max_new_skills=3 控制每次增量，避免一次生成太多低质量技能
  • 所有 dyn_ 前缀的技能在 template 检索时全量返回（优先级高于静态技能）

核心洞察

SkillRL 的”技能”和 self-improving-skills 的”技能”有一个根本区别：

  • SkillRL 的技能是训练数据——注入到 prompt 中帮助 RL 模型学习更好的策略。技能本身不改变模型参数，但通过影响 rollout 来引导策略梯度方向。
  • self-improving-skills 的技能是运行时指令——在 Claude Code 需要时加载，直接改变 agent 的行为。

SkillRL 的动态更新是最接近”自进化”的：

  • 训练过程中 model 在变，skill bank 也在变
  • 两者协同进化：更好的 skills → 更好的 rollout → 更好的梯度 → 更好的 model → 发现新的 failure → 生成新的 skills
  • 这是一个正反馈循环

一句话带走

把 agent 的历史经验蒸馏成 JSON 技能库，在 RL 训练的每一步注入相关技能，当验证失败时自动生成新技能——技能和模型协同进化，形成正反馈循环。 技能是纯数据（不是代码），检索有两种模式（关键词/语义），动态更新有三个 guard（去重、ID 控制、数量限制）。
