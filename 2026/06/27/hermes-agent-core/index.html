Hermes Agent：self-improving-skills 的灵感来源

来源：agent-frameworks/hermes-agent/
团队：Nous Research
语言：Python，MIT 协议，2026年3月发布
源码：github.com/NousResearch/hermes-agent
中文文档：hermes-agent.org/zh

一句话总结

Hermes 是一个全栈 AI Agent 平台（CLI + TUI + 桌面 + 消息网关 + cron + webhook），而 self-improving-skills 把它最核心的技能蒸馏 + curator 循环移植到了 Claude Code 插件生态。Hermes 是本尊，self-improving-skills 是它的一个功能的轻量移植。

设计哲学：窄腰宽边

AGENTS.md 里定义了两个核心原则，贯穿所有设计决策：

1. Prompt Caching 是神圣的

  • 长对话的每一轮都复用一个缓存的 system prompt
  • 任何在对话中途改变 system prompt、交换 toolset、重建上下文的操作都会破坏缓存，成倍增加用户成本
  • 所以不允许这么做

2. 核心窄，能力在边缘

  • 每个新增的 model tool 都会在每次 API 调用中发送

  • 这叫 “The Footprint Ladder”（足迹阶梯）

Skills 系统：另一种部署模型

和 EvoSkill / self-improving-skills 的 SKILL.md 不同，Hermes 的 Skills 是分目录的、有依赖关系的、可链式加载的：

每个 skill 有自己的目录，包含：

  • DESCRIPTION.md（触发条件）
  • SKILL.md 或等效的指令文件
  • 可能的 scripts/、references/ 子目录

Skill 可以链式加载：--skills "arxiv,obsidian" 会把两个 skill 的指令都注入。

和 self-improving-skills 的关系

self-improving-skills 的 README 明确说它是 “Hermes Agent-style self-improvement for Claude Code”，具体移植了：

Hermes 原版 self-improving-skills 移植
Hermes 的 procedural memory + skills ~/.claude/skills/<name>/SKILL.md
Hermes 的 curator loop（7天一次） curator_transitions.py（stale→archive）
Hermes 的 skill_usage telemetry usage_store.py（use/view/patch 计数）
Hermes 的 distiller skill-distiller 子 agent
Hermes 的 SKILLS_GUIDANCE SessionStart 注入的上下文

无法移植的部分（self-improving-skills README 的”局限性”一节）：

Hermes 功能 为什么无法移植
无声免费后台 daemon fork Claude Code 没有对应的进程模型
前缀缓存继承 Claude Code 无此概念
跨 session FTS5 搜索（RECALL） Claude Code API 不支持
内存循环（MEMORY.md） Claude Code 已有原生 auto-memory，故意避开

Hermes 独有的能力

以下是 Hermes 有而我们的五个自进化项目都没有的：

1. Cron + Webhook 集成

2. 多平台投递

3. Script 前置处理

脚本先跑，输出变成 Agent 上下文。脚本做机械工作（fetch/diff/compute），Agent 做推理。

4. Model 无关
同一个自动化，可以用 Claude/GPT/Gemini/DeepSeek/本地模型——按任务成本选择。

5. 你的基础设施上跑
不依赖 Anthropic 的服务器，在自己的 VPS/家用服务器/笔记本上跑。数据在你的机器上。

一句话带走

Hermes 是”能力在边缘”哲学的典范：用窄核心保证 prompt caching 和成本可控，用 Skills + Plugins + MCP + Webhook + Cron 把所有扩展能力推到边缘。 self-improving-skills 是它最核心的”技能蒸馏+curator”子系统的 Claude Code 移植版——功能少得多，但设计精神完全一致。
